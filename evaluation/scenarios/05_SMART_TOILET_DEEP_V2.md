# 시나리오 5-V2 (심층판) — 스마트 토일렛 헬스케어 제품 설계

> **질문**: 가정용 스마트 변기에서 대소변의 냄새·형태를 센서로 인지하고 시그널을 핸드폰으로 전송하는 헬스케어 제품을 만들려 합니다. 어떤 기술·기관·자금이 모여야 하나요?

> **이전 V1 대비**: V1은 정부 R&D 검색 1회·6건만으로 결론을 냈음. V2는 정부 R&D 1,515건(소변 분석)·416건(가스센서 헬스케어)·13건(스마트 변기 특정)·학술 DB 학술 45건·특허 63건·실명 협력 후보 8곳 정밀 매핑으로 재구성.

## 🎯 분석 요약 (Executive Summary)

스마트 토일렛 헬스케어 제품은 단일 기술이 아니라 **9개 기술 도메인 × 4개 부처 × 3단계 인증을 묶는 융합 시스템 통합(SI) 사업**입니다. 한국에는 ETRI(센서 원천기술 14.8억 원/연 사업 보유)·KBSI(광검출 자문)·서울대(비데 장착 가스센서 진행)·**(주)시그마전자펜(국내 최초 스마트 변기 정부 사업 수행)**·㈜아폴론(라만 광검출)·바이오고(전자코)·영동제약(소변 진단) 등 **8개 핵심 협력 후보**가 이미 존재하고, 정부 매칭(2025년 기준 단일 과제 1억~14억 원)이 가능합니다. 다만 한국 시장은 **글로벌 대비 2~3년 뒤처짐**(중국 Xiamen R&T가 특허 다수 보유, 미국 Casana·Toi Labs가 양산 진입)이라 빠른 PoC + 인증 동시 추진이 핵심입니다.

권고: **(A) ETRI 또는 서울대 컨소시엄 합류로 핵심 센서 기술 확보 → (B) 시그마전자펜·아폴론과 라이선스/인수 검토 → (C) 6T 분류 E5(바이오·의료) > E5006(의료정보 및 시스템) > E500602(원격 및 재택 의료기기) 정부 트랙 진입**.

## 🔍 9개 기술 도메인 — 풀스택 분해

### A. 센싱 레이어 (배설물 분석)

**A.1 가스·냄새 센서 (변기 위 5~30cm 공간, 비접촉)**

| 센서 타입 | 대상 분자 | 한국 협력 후보 |
|---|---|---|
| 전기화학식 가스 센서 | NH3(요), H2S(장 가스), CH4(메탄), 인돌·스카톨(대변 특유) | ㈜바이오고 (eNose 감지막), 센코 (시장) |
| 금속산화물 반도체 (MOS) | VOC 총량, 패턴 인식 | ETRI eNose 트랙 |
| 광 이온화 검출 (PID) | VOC 정밀 (ppb 수준) | (수입 위주 — 회피 설계 필요) |
| 적외선 NDIR | CO2, 일부 VOC | 한국전자기술연구원(KETI) 보유 |

**핵심 난관**: VOC는 1,000+ 분자가 섞여 있음 → 의미 있는 5~20개 패턴만 식별해야 함. ETRI의 후각 바이오 정보 알고리즘이 한국 내 최고 수준.

**A.2 광학·분광 센서 (대변 형태·소변 광검출)**

| 센서 | 측정 | 한국 협력 |
|---|---|---|
| CMOS/CCD 카메라 + LED | 형태(브리스톨 척도), 색깔, 부피 | ㈜에스티엠 (CMOS 멀티 리더), LG이노텍 |
| 근적외선 분광 (NIR) | 수분·지방·단백질 비율 | (한국 R&D 적음 — 일본 우위) |
| 라만 분광 (SRS/SERS) | 분자 조성 (요산·요단백·당) | **㈜아폴론**(혈당), KBSI(자문), 피코파운드리 |
| 굴절률·탁도 | 소변 농축도, 잠혈 | 광학 모듈사 |

**브리스톨 척도(Bristol Stool Scale)**: 1~7단계 분류. 학술 DB 검색 결과 "Bristol Stool Scale AI" 단 **4건** — **극히 희소한 분야로 진입 기회 큼**.

**A.3 분석물·생화학 센서 (소변 직접 접촉, 선택적 고급)**

| 센서 | 측정 | 한국 협력 |

| 단백질·잠혈 시험지 (디지털 판독) | 신장 질환 조기 신호 | 영동제약 (급성/만성 신부전증 조기 진단 사업 보유) |
| 미생물·메타게놈 (별도 카트리지) | 장내미생물 | (가정용 단계 어려움) |

### B. 신호 처리 / AI 레이어

**B.1 엣지 AI (변기 내장)**

| 컴포넌트 | 권장 사양 | 비고 |
|---|---|---|
| MCU/SoC | ARM Cortex-A53/A55, 1GB RAM | Raspberry Pi 4, NVIDIA Jetson Nano, Rockchip RK3588 |
| 모델 | TFLite·ONNX 양자화 | 이미지 분류(브리스톨) + e-nose 분류 |
| 처리량 | 1초당 5~10프레임 + 가스 1Hz 샘플링 | 실시간 충분 |

**개인정보 보호 핵심**: 카메라 영상은 엣지에서 분류 후 **즉시 폐기**, 영상 자체는 절대 클라우드 전송 금지. 사용자 수용성 95%↑로 결정적.

**B.2 클라우드 AI**

| 기능 | 알고리즘 |
|---|---|
| 시계열 추적 | LSTM/Transformer — 개인 패턴 학습 |
| 이상치 탐지 | Isolation Forest, AutoEncoder |
| 다중 모달 통합 | Gas + Image + 분석물 결합 (Late fusion) |
| 의료 알림 | 보수적 임계치, False Positive < 5% 설계 |

### C. 통신 / 연결 레이어

```
변기 ─[BLE/WiFi]→ 홈 게이트웨이 ─[5G/광인터넷]→ 클라우드 ─[Push]→ 핸드폰 앱
                                                       │
                                                       └→ 의료기관 API (선택)
```

| 단계 | 프로토콜 | 한국 협력 |
|---|---|---|
| 변기 → 게이트웨이 | BLE 5.0(저전력) 또는 WiFi 6 | 텔레칩스, 라온피플 |
| 게이트웨이 → 클라우드 | MQTT over TLS | AWS IoT, NHN Cloud, 네이버클라우드 |
| 클라우드 → 핸드폰 | APNs/FCM Push | 표준 |
| 의료기관 연동 | FHIR(국제 표준), HL7 | 카카오헬스케어, 네이버헬스케어 |
